# Introduction

The 0x API provides a unified interface for liquidity aggregation, token swap pricing, and onchain trading across major supported blockchains. It is designed to help developers easily integrate token swaps and routing logic without managing individual DEX integrations.

The 0x API exposes endpoints for the following core capabilities:

- **[Swap](/api-reference/openapi-yaml/swap/allowanceholder-getprice)**: Generate executable quotes and calldata for token swaps using either the AllowanceHolder (recommended) or Permit2 approval models. Learn more in the [FAQ](/docs/introduction/faq#allowanceholder-and-permit2).
- **[Gasless](/api-reference/openapi-yaml/gasless/getprice)**: Enable gasless swaps by sponsoring transaction fees on behalf of users, improving UX for end-user applications.
- **[Sources](/api-reference/openapi-yaml/sources/getsources)**: Retrieve supported liquidity sources, chains, and token metadata used by the 0x routing engine.
- **[Trade Analytics](/api-reference/openapi-yaml/trade-analytics/trade-analytics-swap)**: Query historical trade data, volumes, and execution details for monitoring and analytics use cases.

Each API surface is documented in detail in the sections below.

## Base URL

All requests to the 0x API must be made to the following base URL:

```
https://api.0x.org

```

All requests must be sent over HTTPS. Requests made over HTTP will be rejected.

## Authentication

All 0x API requests require an API key. Include your API key in the following request header:

<ParamField header="0x-api-key" type="string" example="YOUR_API_KEY" required>
  Your 0x API key used to authenticate requests.
</ParamField>

<ParamField header="0x-version" type="string" example="v2" required>
  0x API version.
</ParamField>

Requests missing this header may be rate-limited or rejected.

<Info>
  You can generate and manage API keys from the [0x
  Dashboard](https://dashboard.0x.org).
</Info>

## Chain Support

The 0x API supports multiple EVM-compatible chains. Each request must specify a chain either via the request path or query parameters, depending on the endpoint.

Refer to the [Supported Chains](/docs/introduction/supported-chains) for a complete and up-to-date list.

  <Tab title="JavaScript">
    ```javascript theme={"system"}
    const params = new URLSearchParams({
      sellAmount: "100000000000000000000000", // 100,000 WETH
      taker: "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045", // vitalik.eth
      chainId: "1",

    });

    fetch(`https://api.0x.org/swap/allowance-holder/quote?${params.toString()}`, {
      headers: {
        "0x-api-key": "YOUR_API_KEY",
        "0x-version": "v2",
        "Content-Type": "application/json",
      },
    })
      .then((res) => res.json())
      .then((data) => console.log(data));
    ```

  <Tab title="Python">
    ```python theme={"system"}
    import requests

    url = "https://api.0x.org/swap/allowance-holder/quote"
    params = {
        "sellAmount": "100000000000000000000000",  # 100,000 WETH
        "taker": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",  # vitalik.eth
        "chainId": "1",
        "sellToken": "0xc02aaa39b223fe8d0a0e5c4f27ead9083c756cc2",  # WETH
        "buyToken": "0x6b175474e89094c44da98b954eedeac495271d0f",   # DAI
    }

    headers = {
        "0x-api-key": "YOUR_API_KEY",
        "0x-version": "v2",
        "Content-Type": "application/json",
    }

    response = requests.get(url, params=params, headers=headers)
    data = response.json()
    print(data)
    ```
